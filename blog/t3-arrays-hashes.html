Arrays and Hashes

Collections in Ruby.

June 30, 2015

Arrays and hashes in Ruby are useful collections of objects or variables that you can easily access when needed. Both are useful though they operate slightly differently. Let's take a look at these differences so you can determine which one is best for your program.

Like a virtual filing cabinet where each drawer holds information that you have determined, arrays hold variables, integers, or strings. To initilize an array we call it equal to empty brackets. 

myArray = []
This array is empty and waiting for you to fill it with elements. 
myArray = [1, 5.5, "ten"]
This array contains an integer, 1, a float, 5.5, and a string, "ten". To access these elements, you call the location number, think of each drawer in your filing cabinet having a number on the door. 
myArray [0]
The first element is always 0 so if you have an array of 4 different integers, you have elements 0 through 3.

Hashes, on the other hand, will hold exactly what an array can but instead of the doors of your vitual filing cabinet having a predetermined number you must assign each location a label or key. 

myHash = {
           'strawberry'   =>  'red',
           'blueberry'    =>  'blue',
           'banana'       =>  'yellow',
           'watermelon'   =>  'red'
           }
Here we have mapped fruits based on their color. Now if you call the color red you'll get strawberry and watermelon. This can be useful when dealing with multiple objects.

Happy coding!
