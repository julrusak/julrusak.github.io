Ruby Versus JavaScript

The same, but different.

August 1, 2015

JavaScript and Ruby have much of the same qualities, which makes it easier to understand one if you know the other. They both accept integers, strings, booleans, and floats, among other objects. They both deal with arrays and hashes. They both have built-in methods or functions and BONUS some of them are abbreviated the same way! Like .push to add a value into an array and .map to iterate through an array! Some others are .reverse, to reverse an array or string, and .slice to remove a section of an array. This is super helpful when you are starting out, like I am. I find that I still have to look up a few key terms and the syntax is a bit different, but as a newbie, I can rely on some staple functions in my programs. Keep in mind that for each method called in JavaScript, you must follow it with (). Inside may go arguments or it may be left blank, but you need the parentheses. So let's look at some syntax and break down Ruby classes vs. JavaScript constructor functions!

A Ruby class contains methods, like so; 

        class Human
          def initialize (name, location)
            @name = name
            @location = location
          end

          def greeting
            puts "Hi #{@name}!"
          end

          def small_talk
            puts "How's the weather in #{@location}?"
          end
Fairly intuitive. You can understand exactly what is going on because Ruby is incredibly reader and user friendly. Let's look at the same in JavaScript. 

        
          var human = {}
          human.greeting = function(name){
          return "Hi " + name + "!";
        }
          human.smallTalk = function(location){
          return "How's the weather in " + location + "?";
        }
This is, of course, only one way to write this as JavaScript accepts many ways of determining objects. But as you can see, there are less lines of code but more punctuation.

While I am just starting out with both languages, I did form a quick bond with Ruby, which I didn't with JavaScript. Both languages make sense to me and both are very necessary as a developer. Perhaps in future blog posts I have a stronger opinion for either! Until then, keep learning!

Happy Coding!
