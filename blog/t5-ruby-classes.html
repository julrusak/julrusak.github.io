Classes in Ruby

When and Why?

July 18, 2015

When I first learned about classes in coding, it was in a course titled Creative Coding. The nature of the course was to learn how to design and animate using code. So we learned how to draw a lot of shapes, then we learned how to animate these shapes by adding 1 to their X and Y coordinates using loops. Using this method was super easy when you just want to make a circle float up the screen, but what if you want to make a game with a series of bubbles that dissapear when you touch them? You can animate each bubble, but that would waste a lot of time, so this is where you would use a class!

A class will create an object that can be used over and over. Within the class you can define a series of methods which tell your object what to do. Creating a bubble game can be as simple as this; 

        class Bubble
          def float
            #code that will animate an elipse to rise
          end

          def pop
            #code that will make the bubble disappear when the mouse clicks on the ellipse.
          end
After you create a new bubble, you can call all of the methods on the bubble. For example; 
        bubbleOne = Bubble.new
        bubbleOne.float
        bubbleOne.pop

However, there's a huge problem in my example. Ruby is a back-end language. Meaning we would create shapes and animate in a different language altogether. I just wanted to use that example because it made so much sense to me when learning classes. Classes are great when you want a game with tons of bubbles, then you don't have to code out a hundred bubbles, you make a class which acts like a container holding as many bubbles as you will eventually call for, complete with all the behaviors you need. So when would you use classes in Ruby? Anytime you will call an object more than once. Maybe you are buiding a calculator so you would want a math class. 

        class Calculator
          def addition (x, y)
            return x + y
          end

          def subtraction (x, y)
            return x - y
          end
        end
You can go on and on with the math methods and now you have built a calculator!

Happy Coding!
